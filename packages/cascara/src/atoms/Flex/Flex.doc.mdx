The Flex component will create a container with the css flex properties.

## Usage

### Default

```jsx
<Flex>
  <Button content='Flex' />
</Flex>
```

### Space Start

```jsx
<Flex space='start'>
  <Button content='Button' />
  <Button content='Inside' />
  <Button content='Flex' />
  <Button content='Component' />
</Flex>
```

### Space Start Push

It creates a menu button with the content specified inside the Flex.Item component

```javascript
<Flex space='start'>
  <Button content='Button' />
  <Button content='Inside' />
  <Button content='Flex' />
  <Button content='Component' />
  <Flex.Item push>
    <ActionStack actions={[{ label: 'Stuff' }, { label: 'More Stuff' }]} />
  </Flex.Item>
  <Button content='Button' />
</Flex>
```

### Space End

```jsx
<Flex space='end'>
  <Button content='Button' />
  <Button content='Inside' />
  <Button content='Flex' />
  <Button content='Component' />
</Flex>
```

---

This component will eventually go back to Cascara. We can probably safely export it atomically as well. Instead of everyone having to write the same styles over and over again for the correct flexbox structures everywhere, we can just a have component wrapper to do the same thing. This should allow us to more quickly arrange our components in rows or columns without trying to share styles everywhere for consistency.

# Example

```jsx
---
title: An example use of Flex.
---
<Flex>
  <div key='Column 1'>Column 1</div>
  <div key='Column 2'>Column 2</div>
</Flex>
```

> The children count must match the `count` prop, otherwise the layout will be broken.

This action module renders a simple button, use it for custom actions.

## Using the module

Additional to the props specified aside, please use the following property to configure your module:

| property | type                | description                                           |
| -------- | ------------------- | ----------------------------------------------------- |
| module   | required **String** | specifies the type of module, in this case **button** |

> Adding this module to your **Cascara** **Table** or **Form** would look like this:

## Example

Adding this module to a **Cascara** `Table` would look like this:

```jsx
<Table
  // define the actions we want to appear for each table row
  actions={{
    actionButtonMenuIndex: -1,
    modules: [
      {
        module: 'button',
        name: 'test',
        content: 'Test',
        size: 'small',
      },
    ],
  }}
  // your data
  data={[
    {
      id: 1,
      userEmail: 'john@doe.it',
      userName: 'John Doe',
    },
  ]}
  // the fields in our data you want to display
  dataConfig={{
    display: [
      {
        attribute: 'id',
        isEditable: false,
        isLabeled: false,
        label: 'ID',
        module: 'number',
      },
      {
        attribute: 'userEmail',
        isEditable: false,
        isLabeled: false,
        label: 'eMail',
        module: 'text',
      },
      {
        attribute: 'userName',
        isEditable: false,
        isLabeled: false,
        label: 'Name',
        module: 'text',
      },
    ],
  }}
  // the action handler is invoked when a button is clicked
  onAction={(action, record) => {
    console.clear();
    console.log(`Action ${action.name} invoked`);
    console.table(record);

    switch (action.name) {
      case 'test': // this has to be the same as your action name
        console.log(`Your action '${action.name}' has been invoked!`);
        break;
    }
  }}
  // this defines the unique ID field in your data
  uniqueIdAttribute={'id'}
/>
```

> The above example renders a `Table`, this is because modules can only be used inside `Table` of `Form`.

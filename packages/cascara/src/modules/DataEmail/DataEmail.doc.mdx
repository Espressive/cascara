A data module to display `email` values, it renders a input of type `email`.

## Email shape

Use the following properties to configure your `email` data module:

| property   | type               | description                                                                                               |
| ---------- | ------------------ | --------------------------------------------------------------------------------------------------------- |
| attribute  | required `String`  | the attribute name in your data object you want this module to display                                    |
| module     | required `String`  | specifies the type of module, in this case `email`                                                        |
| isEditable | optional `Boolean` | specifies if the module should render the edition markup when parent enters in `edit` mode                |
| isLabeled  | optional `Boolean` | specifies if the module should render a lable next to it (useful in `Form`, set it to `false` in `Table`) |
| label      | optional `String`  | the label text to display if a label is rendered                                                          |

module

Adding this module to your **Cascara** `Table` or `Form` would look like this:

```jsx

---
title: A Table displaying an `email` module
---

<Table

  // we need to enable edition by passing the `edit` module
  actions={{
    actionButtonMenuIndex: -1,
    modules: [
      {
        module: 'edit',
        cancelLabel: 'Cancelar',
        editLabel: 'Editar',
        saveLabel: 'Guardar',
      },
    ],
  }}

  // your data
  data={[
    {
      id: 1,
      email: 'test@example.com',
    },
  ]}

  // the fields in our data you want to display
  dataConfig={{
    display: [
      {
        attribute: 'email',
        isEditable: true,
        isLabeled: false,
        label: 'eMail',
        module: 'email',
      },
    ],
  }}

  // the action handler is invoken when a button is clicked
  onAction={(action, record) => {
    console.clear();
    console.log(`Action ${action.name} invoked`);
    console.table(record);

    switch (action.name) {
      case 'edit.start':
        console.log('Entering edit mode');
        break;
      case 'edit.cancel':
        console.log('Exiting edit mode, edit canceled');
        break;
      case 'edit.save':
        console.log('Exitin edit mode, saving changes');
        break;
    }
  }}

  // this defines the unique ID field in your data
  uniqueIdAttribute={'id'}
/>
```
